CuriosityAI identifies unexplored research areas by analyzing density distributions in embedding space. The system fetches papers from ArXiv and PubChem APIs, transforms them into high-dimensional embeddings, then applies GMM or KDE to detect low-density regions representing conceptual gaps. Autonomous agents evaluate these gaps for feasibility and generate structured research proposals with GitHub-hosted prototype code.

## Gaussian Mixture Model Gap Detection

The core algorithm models the distribution of existing research papers using scikit-learn's GMM in embedding space. Low-probability density regions indicate areas with minimal prior work, representing potential invention opportunities. The system ranks gaps by computing probability scores at sampled coordinates throughout the embedding manifold, filtering noise using configurable density thresholds. Vec2Text from Hugging Face performs embedding inversion to generate natural language descriptions of the unexplored concepts represented by low-density coordinates.

## Technical Features

- **Multi-Agent Orchestration** — FetchAI and LangChain agents coordinate parallel workflows for gap analysis, with specialized agents handling feasibility evaluation, proposal drafting, and code generation tasks independently.

- **RAG Pipeline** — ChromaDB vector store enables semantic search across research papers and patents, retrieving relevant context that agents inject into Claude or Groq LLM prompts for grounded proposal generation.

- **Embedding Inversion** — Vec2Text transformer architecture reconstructs text from embedding vectors, translating numerical gap coordinates into coherent research problem statements without requiring reverse lookup in the corpus.

- **Voice Interface** — Deepgram speech-to-text integration enables researchers to verbally query the system and receive spoken summaries of discovered gaps and generated proposals.

- **3D Visualization** — Three.js renders interactive embedding space visualizations in the Next.js frontend, displaying papers as points and highlighting low-density gap regions with color-coded probability contours.

- **Automated GitHub Integration** — Agents generate prototype implementations based on proposals and use GitHub API to create repositories, commit code with structured messages, and initialize README documentation.

## Architecture

The Python backend implements a pipeline architecture where ingestion modules fetch papers via ArXiv and PubChem APIs, batch-process them through Hugging Face sentence transformers, and store vectors in ChromaDB with metadata indexing. The gap detection module runs periodic GMM fitting on the embedding corpus and samples the probability density function to identify candidate low-density regions. FetchAI agents operate as independent workers that query ChromaDB for context, invoke LLMs for analysis and proposal generation, then interface with GitHub API for code deployment. The Next.js frontend connects via REST endpoints to retrieve gap analysis results and streams them to Three.js for real-time visualization.
